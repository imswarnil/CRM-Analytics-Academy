# Interactivity & Actions

A great dashboard doesn't just display numbers — it lets users dig in and act. This lesson covers the interactive features that elevate a dashboard from report to tool.

## Faceting and selections

By default, charts on the same dataset **facet** each other: click a bar and every related chart updates. Tune this behavior per widget:

- Turn faceting **off** for context charts that should stay fixed.
- Use **selection bindings** for cross-dataset interactions.
- Set a **default selection** so the dashboard opens on a useful view.

## Drill-downs and navigation

Give users a path deeper into the data:

- **Drill into a record** — jump from a chart to the underlying Salesforce record.
- **Open another dashboard** — pass the current selection as a filter via bindings.
- **Page navigation** — split a busy dashboard into tabs/pages users step through.

::note
Passing selections between dashboards uses the same binding functions you learned in the SAQL module — `cell(...).asString()` to carry a value into the destination's filters.
::

## Conditional formatting

Color communicates faster than numbers. Apply conditional formatting to:

- Turn KPIs **red/green** against a target.
- Shade table cells by value (heatmap).
- Highlight bars that cross a threshold.

::tip
Anchor formatting to a **business target**, not an arbitrary number. "Below quota" is meaningful; "below 50,000" usually isn't.
::

## Salesforce actions

CRM Analytics can trigger Salesforce actions directly from a dashboard:

- **Update a record** — change a field on the selected row.
- **Create a record** — log a task or case.
- **Open a record action** — run a quick action or flow.

This closes the loop: see the insight and act on it without leaving the dashboard.

## Embedding in Lightning

Dashboards are most powerful in context. Embed them with the **CRM Analytics Dashboard** Lightning component:

- On a record page, filtered to that record.
- On a Lightning app page as a landing analytics hub.
- In a Community/Experience Cloud site for external users.

## Performance tips

Keep interactive dashboards fast:

- Limit the number of queries per page (aim for under ~10).
- Pre-aggregate heavy data in a recipe instead of at query time.
- Avoid very high-cardinality groupings in the initial view.
