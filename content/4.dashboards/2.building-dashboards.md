# Building Dashboards

A **dashboard** combines multiple queries and widgets into a single interactive page. This lesson covers the designer, the widget toolkit, and how to lay out a dashboard that reads well.

## The dashboard designer

Create a dashboard with **Create → Dashboard**. The designer has three areas:

- The **canvas** where you place widgets.
- The **widget toolbar** on the left.
- The **query panel** at the bottom listing every query the dashboard runs.

You drag widgets onto the canvas and bind them to queries.

## Essential widgets

A handful of widgets cover most needs:

- **Chart** — visualizes a query (bar, line, etc.).
- **Number** — a single KPI value.
- **Table / Values** — detailed rows.
- **List / Toggle / Dropdown** — let users filter.
- **Date** — relative or absolute date selection.

::note
Filter widgets (list, dropdown, date) don't show data themselves — they exist to drive **bindings** into your chart queries.
::

## Layouts and responsiveness

CRM Analytics dashboards support multiple layouts so one dashboard adapts to screen size:

- Build a **desktop** layout first.
- Add a **mobile** layout that reuses the same queries with a stacked arrangement.
- Use the layout's **device targeting** to control which renders where.

This is how a single dashboard looks right on both a laptop and the Salesforce mobile app.

## Structuring for readability

Guide the eye top to bottom:

1. **KPIs** (number widgets) across the top — the headline metrics.
2. **Trends** (line charts) in the upper middle.
3. **Breakdowns** (bar charts, tables) below.
4. **Filters** in a consistent place (top or left).

::tip
Leave whitespace and align widgets to a grid. A cluttered dashboard erodes trust in the numbers, no matter how accurate they are.
::

## Global filters and faceting

**Faceting** automatically links charts built on the same dataset — selecting in one filters the others with no manual bindings. For cross-dataset filtering, add explicit bindings (from the SAQL module).

Enable or disable faceting per widget in its properties when you need finer control.

## Saving and sharing

Save the dashboard into an app, then share the **app** with Viewer access for consumers. Preview it with **Run As** to confirm row-level security behaves before you hand it to the business.
