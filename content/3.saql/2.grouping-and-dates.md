# Grouping, Dates & Windows

Real analysis needs more than a single group-by. This lesson covers the SAQL features that turn flat aggregates into rich, time-aware insight.

## Grouping by multiple dimensions

Group by several fields to create a matrix:

```sql
q = group q by ('Region', 'StageName');
q = foreach q generate
      'Region' as 'Region',
      'StageName' as 'Stage',
      sum('Amount') as 'Pipeline';
```

The result has one row per Region/Stage combination — perfect for a stacked bar or pivot.

## Working with dates

Dates in datasets expand into parts you reference by name:

```sql
q = group q by ('CloseDate_Year', 'CloseDate_Quarter');
q = foreach q generate
      'CloseDate_Year' as 'Year',
      'CloseDate_Quarter' as 'Quarter',
      sum('Amount') as 'Revenue';
```

Use relative ranges in filters for moving windows:

```sql
q = filter q by date('CloseDate_Year', 'CloseDate_Month', 'CloseDate_Day')
      in ["1 year ago".."current day"];
```

## Filtering on aggregates

To filter *after* aggregation (like SQL's `HAVING`), group, project, then filter:

```sql
q = group q by 'AccountName';
q = foreach q generate 'AccountName' as 'Account', sum('Amount') as 'Total';
q = filter q by 'Total' > 100000;
q = order q by 'Total' desc;
```

::note
Order matters. Filtering before the `foreach` filters rows; filtering after it filters **groups** by their aggregated values.
::

## Windowing functions

Windows compute values across rows without collapsing them — running totals, rankings, and percent-of-total:

```sql
q = group q by ('Region', 'CloseDate_Month');
q = foreach q generate
      'Region' as 'Region',
      'CloseDate_Month' as 'Month',
      sum('Amount') as 'Revenue',
      sum(sum('Amount')) over ([..0] partition by 'Region' order by 'CloseDate_Month') as 'RunningTotal';
```

The `over (...)` clause defines the window frame and partition.

## Percent of total

A common windowed pattern is share-of-total:

```sql
q = foreach q generate
      'Region' as 'Region',
      sum('Amount') as 'Revenue',
      sum('Amount') / sum(sum('Amount')) over ([] partition by all) as 'PctOfTotal';
```

::tip
`partition by all` with an empty frame `[]` gives the grand total — divide by it to get each row's percentage.
::
