# SAQL Fundamentals

**SAQL** (Salesforce Analytics Query Language) is the query language behind every lens and dashboard. The UI writes SAQL for you, but learning it directly unlocks advanced calculations and debugging.

## The query pipeline

SAQL reads like a pipeline. Each statement takes a stream, transforms it, and assigns the result back to a variable — conventionally `q`:

```sql
q = load "Opportunities";
q = filter q by 'StageName' == "Closed Won";
q = group q by 'Region';
q = foreach q generate 'Region' as 'Region', sum('Amount') as 'Revenue';
```

Each line refines the previous result. The final `q` is what gets returned.

## Loading data

Every query begins by loading a dataset:

```sql
q = load "Opportunities";
```

The string is the dataset's **API name**, not its label. You can load and join multiple datasets in advanced queries, but most start with a single load.

## Filtering rows

The `filter` statement keeps only matching rows:

```sql
q = filter q by 'Amount' > 10000;
q = filter q by 'StageName' in ["Negotiation", "Proposal"];
q = filter q by date('CloseDate_Year', 'CloseDate_Month', 'CloseDate_Day') in ["current quarter"];
```

::tip
Filter as **early** as possible in the pipeline. Reducing the row count before grouping makes queries faster and cheaper.
::

## Grouping and aggregating

`group` defines the buckets; `foreach ... generate` produces the output columns:

```sql
q = group q by 'StageName';
q = foreach q generate
      'StageName' as 'Stage',
      count() as 'Deals',
      sum('Amount') as 'Pipeline';
```

Common aggregate functions include `sum()`, `avg()`, `count()`, `min()`, `max()`, and `unique()`.

## Ordering and limiting

Shape the final result with `order` and `limit`:

```sql
q = order q by 'Pipeline' desc;
q = limit q 10;
```

This returns the top 10 stages by pipeline — a classic "top N" pattern.

## Projecting without grouping

You can also use `foreach` without grouping to compute row-level columns:

```sql
q = foreach q generate 'Name' as 'Account', 'Amount' * 0.1 as 'EstCommission';
```

::note
`foreach` always defines the output columns. After a `group`, it runs once per group; without a `group`, it runs once per row.
::
