# Setup & Access

Before you can build anything, CRM Analytics has to be enabled and your users need the right permissions. This lesson walks through the one-time setup.

## Licensing basics

CRM Analytics is a paid add-on to Salesforce. The license you have determines what you can do:

- **CRM Analytics Growth** — full build and view capabilities for analysts and admins.
- **CRM Analytics Plus** — Growth plus higher Einstein Discovery limits and more rows.
- **Embedded / view-only** — consume dashboards embedded in Lightning, without building.

::warning
Row and concurrency limits differ by license. Check **Setup → Company Information** and your contract before designing very large datasets — hitting a row cap mid-project is a common surprise.
::

## Enabling CRM Analytics

An admin enables the platform once:

1. Go to **Setup → Analytics → Getting Started**.
2. Click **Enable CRM Analytics**.
3. Wait for provisioning to finish (usually a few minutes).

Once enabled, the **Analytics Studio** app appears in the App Launcher.

## Permission sets that matter

Salesforce ships standard permission sets you assign to users:

- **CRM Analytics Plus Admin** — full administrative control.
- **CRM Analytics Plus User** — build datasets, lenses, and dashboards.
- **CRM Analytics Plus Viewer / Embedded App User** — view-only access.

Assign them under **Setup → Permission Sets → (select) → Manage Assignments**.

::tip
Create a custom permission set group that bundles the analytics permission set with your other internal permissions. It makes onboarding new analysts a one-click task.
::

## Key system permissions

Within those permission sets, a few flags control important capabilities:

- **Manage CRM Analytics** — create apps, manage all assets.
- **Create CRM Analytics Dashboards** — build and edit dashboards.
- **Use CRM Analytics Templated Apps** — install prebuilt analytics apps.
- **Edit Dataset Recipes** — build and run recipes.

## Verifying your access

To confirm everything is wired up:

1. Open the **App Launcher** and search for **Analytics Studio**.
2. You should land on the Studio home with tabs for **Browse**, **Datasets**, and **Recipes**.
3. If you can click **Create → App**, you have build permissions.

If any step fails, revisit the permission set assignments before moving on.
