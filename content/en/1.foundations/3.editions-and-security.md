# Editions, Licensing & Security

CRM Analytics is a separate product from core Salesforce, so before you build anything it helps to know how access is granted and how data is protected once it lands in a dataset.

## Licenses you'll run into

- **CRM Analytics Plus** — full platform access: recipes, dataflows, dashboards, and Einstein Discovery.
- **CRM Analytics Growth** — a lighter tier aimed at smaller teams, with usage limits on data rows and users.
- **Permission set licenses** — individual users are granted access through permission sets layered on top of the org license, so not everyone in an org automatically sees CRM Analytics.

## Row-level security

Datasets don't inherit Salesforce object permissions automatically. Security is defined explicitly, usually one of:

- **Inherited sharing** — reuse the security of the source Salesforce object (e.g. Opportunity sharing rules).
- **Predicate-based security** — a filter expression evaluated per user (for example, restricting rows to a user's own region) applied at the dataset level.
