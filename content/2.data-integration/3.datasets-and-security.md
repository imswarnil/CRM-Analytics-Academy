# Datasets & Row-Level Security

A **dataset** is the optimized table that powers every lens and dashboard. Understanding its structure and how to secure it is essential before you start visualizing.

## Dimensions, measures, and dates

CRM Analytics types every field into one of three roles:

- **Dimensions** — text/categorical fields you group and filter by (Stage, Region, Owner).
- **Measures** — numeric fields you aggregate (Amount, Quantity, Score).
- **Dates** — special fields enabling time-based grouping and relative filtering.

::note
The same source column can only play one role. If a numeric code is really a category (like a 4-digit region code), convert it to a dimension in your recipe.
::

## Why dataset structure matters

Because data is stored columnar and in-memory:

- Grouping by a **dimension** is extremely fast.
- Aggregating a **measure** is what the engine is optimized for.
- **High-cardinality** dimensions (millions of unique values) cost more memory — model them carefully.

Good structure decisions in the recipe pay off in every dashboard.

## Working with dates

Date fields unlock the most powerful filtering in the platform:

- Group by **year, quarter, month, week, or day**.
- Use **relative ranges** like "current quarter" or "last 90 days".
- Compute **day differences** for cycle-time analysis.

A dataset can hold several date fields (Created, Closed, Last Activity) — each independently groupable.

## Security predicates

A **security predicate** is a row-level filter automatically applied to every query a user runs against the dataset. It answers: *which rows is this user allowed to see?*

A predicate is a single expression, for example:

```text
'OwnerId' == "$User.Id"
```

This row-level rule means each user only sees records they own.

## Common predicate patterns

- **Owner-based**: `'OwnerId' == "$User.Id"` — see only your records.
- **Role hierarchy**: combine with a role lookup so managers see their team.
- **Public flag**: `'IsPublic' == "true" || 'OwnerId' == "$User.Id"`.

::warning
Predicates reference fields that must exist **in the dataset**. If you secure by owner, make sure the recipe outputs an `OwnerId` column. A typo in a predicate can hide all rows.
::

## Testing security

Always verify a predicate with **Run As** or by logging in as a test user. Confirm that:

- The right rows appear for a standard user.
- Admins or managers see the broader set they should.

With secured, well-structured datasets in place, you are ready to query them with SAQL in the next module.
