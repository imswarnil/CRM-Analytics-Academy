# Building Recipes (Data Prep)

A **recipe** is CRM Analytics' modern, visual ETL tool. You start from synced data and add **nodes** that transform it step by step, ending in one or more output datasets.

## Anatomy of a recipe

Every recipe is a graph of nodes flowing left to right:

- **Input** — the source data (a synced object or another dataset).
- **Transform** — clean, derive, and reshape columns.
- **Join** — combine two streams on a key.
- **Aggregate** — roll data up to a coarser grain.
- **Output** — write the result to a dataset.

You build it on a canvas, previewing data at every node.

## Common transform nodes

Inside a Transform node you can:

- **Edit attributes** — rename, retype, and format columns.
- **Create formula fields** — derived columns using functions.
- **Bucket** values into categories (e.g. deal sizes into Small/Medium/Large).
- **Split, trim, and clean** text.

::tip
Do data cleanup as early as possible in the recipe. Fixing a messy column once at the top keeps every downstream node simpler.
::

## Joining data

The **Join** node blends two streams. Pick the join type deliberately:

- **Lookup** — enrich rows with matching columns from another stream (most common).
- **Left / Right** — keep all rows from one side.
- **Inner** — keep only matching rows.

A typical example: join Opportunities to Users to bring the owner's region onto every deal.

## Aggregating to the right grain

The **Aggregate** node summarizes data — for example, total pipeline per account:

```text
Group by:  AccountId
Measures:  sum(Amount) → TotalPipeline
           count()     → OpenDeals
```

::warning
Aggregation changes the **grain** of your data. After an aggregate, one row represents a group, not an individual record. Keep a separate detail dataset if you still need row-level analysis.
::

## Outputs and registration

The **Output** node writes a dataset. You choose:

- The **dataset name** and the **app** it lives in.
- Whether to **replace** or **append** rows.
- The **security predicate** (covered in the next lesson).

A single recipe can have multiple outputs — for example, a detailed dataset and a pre-aggregated summary.

## Running and scheduling

Click **Save & Run** to execute the recipe once, then schedule it to keep outputs fresh. Watch the run in the **Monitor** tab and confirm the row count looks right before building dashboards on top of it.
